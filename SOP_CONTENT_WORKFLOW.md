## 一、内容分类体系

### 1.1 分类（categories）

```
基础教程    - Python、数学、ML 入门
深度学习    - 框架、架构、原理
计算机视觉  - CV 相关
自然语言处理 - NLP 相关
```

### 1.2 标签（tags）规则

每篇文章 3-6 个标签，遵循：**领域 + 细分 + 难度 + 工具/框架**

例：
```
深度学习, 神经网络, 入门教程, AI基础, PyTorch
计算机视觉, 图像分类, CNN, 实战项目, ResNet
NLP, Transformer, BERT, 进阶教程
```

## 二、内容发布流程

### 2.1 单篇文章生产 SOP（5 步）

```
┌─────────────────────────────────────────────────┐
│ Step 1: 选题                                     │
│   ↓ 从选题池选取，确认分类/标签/难度               │
│ Step 2: 写作                                     │
│   ↓ 按模板起草，确保代码可运行、数据可复现          │
│ Step 3: 构建                                     │
│   ↓ hexo clean && hexo generate，验证无报错         │
│ Step 4: 预览                                     │
│   ↓ hexo server 本地预览，检查排版/代码高亮/图片     │
│ Step 5: 发布                                     │
│   ↓ git commit → git push → Cloudflare 自动部署    │
└─────────────────────────────────────────────────┘
```

### 2.2 Front Matter 模板

```yaml
---
title: [文章标题，简洁有力]
date: YYYY-MM-DD HH:mm:ss
categories:
  - [分类，从 1.1 体系选取]
tags:
  - [标签1]
  - [标签2]
  - [标签3]
  - [难度标签]
cover: /img/posts/[封面图].png   # 可选，有则填
description: [100字以内摘要，用于 SEO]
---
```

### 2.3 文章结构规范

```
## 引言（1-2段）
   - 为什么学这个？解决什么问题？
   - 用场景/故事/数据开头，降低阅读门槛

## 前置知识（如需要）
   - 读者需要掌握什么
   - 引用站内已有文章（内链）

## 核心内容（主体）
   - 代码块标注语言（python/py/tensorflow等）
   - 关键步骤配图说明
   - 重要概念加粗标注

## 实战/动手环节
   - 可运行的代码示例
   - 预期输出结果截图
   - 常见问题 FAQ

## 总结
   - 学到了什么
   - 下一步推荐阅读（站内链接）
```

## 四、文章写作规范

### 4.1 代码规范

```python
# ✅ 正确：有注释，有导入，可独立运行
import torch
import torch.nn as nn

# 定义简单的神经网络
class SimpleNet(nn.Module):
    """三层全连接网络，用于 MNIST 分类"""
    def __init__(self):
        super().__init__()
        self.fc1 = nn.Linear(784, 128)
        self.fc2 = nn.Linear(128, 64)
        self.fc3 = nn.Linear(64, 10)

    def forward(self, x):
        x = torch.relu(self.fc1(x))
        x = torch.relu(self.fc2(x))
        return self.fc3(x)
```

### 4.2 图片规范

- 存放路径：`source/img/posts/{文章名}/`
- 格式：PNG（截图）、SVG（示意图）
- 宽度：不超过 800px
- 每篇文章配图至少 1 张（封面/流程图/效果对比）

### 4.3 SEO 规范

- title：包含核心关键词，15-30 字
- description：100 字以内，包含核心关键词
- 首段包含文章主关键词
- 合理使用 h2/h3 层级结构

### 4.4 内链规范

- 引用站内文章使用相对路径或站内搜索链接
- 每篇文章至少引用 2 篇站内相关文章
- 系列文章之间相互链接（上一篇/下一篇）

## 五、发布与部署 SOP

### 5.1 发布检查清单

```
□ Front Matter 完整（title/date/categories/tags/description）
□ 代码块标注语言
□ 图片路径正确且已放入 source/img/
□ 内链检查（引用的站内文章确实存在）
□ hexo generate 无报错
□ 本地预览排版正常
□ 移动端适配检查
□ commit message 符合规范
```

### 5.2 发布命令

```bash
# 1. 构建验证
cd aijvs-blog
npx hexo clean && npx hexo generate

# 2. 本地预览（可选）
npx hexo server

# 3. 提交推送
git add -A
git commit -m "feat: 新增《文章标题》"
git push origin main

# 4. 验证（Cloudflare Pages 自动构建，约1-2分钟）
# 访问 https://aijvs.com 确认
```

### 5.3 紧急修复流程

```bash
# 直接修改 → 推送 → 自动部署，无需额外操作
git add -A
git commit -m "fix: 修复《XX》文章的XX问题"
git push origin main
```

## 六、质量标准

### 6.1 文章质量检查点

- [ ] 零基础读者能看懂吗？（🟢 级别文章）
- [ ] 代码在 Python 3.10+ / PyTorch 2.x 下能跑通吗？
- [ ] 有至少 1 个完整的可运行示例吗？
- [ ] 预计阅读时间 5-15 分钟
- [ ] 不存在过时的 API 调用或废弃写法

### 6.2 禁止事项

- ❌ 不搬运/抄袭他人文章（可参考，但必须重写）
- ❌ 不发布付费内容（全部免费）
- ❌ 不使用未经授权的图片（优先用代码生成或 CC 授权）
- ❌ 不在文章中放置广告

## 七、站点运维 SOP

### 7.1 日常运维

| 事项 | 频率 | 说明 |
|------|------|------|
| 检查网站可访问性 | 每日 | ping aijvs.com + 浏览器访问 |
| 检查 Cloudflare Pages 构建状态 | 发布后 | GitHub push 后确认构建成功 |
| 检查 npm 依赖安全 | 每月 | `npm audit`，修复高危漏洞 |
| 备份内容 | 每月 | Git 仓库即备份，确保 push 成功 |

### 7.2 GitHub 仓库管理

- 主分支：`main`（唯一部署分支）
- 不要提交 `db.json`、`public/`、`node_modules/`
- `.gitignore` 已配置，新增忽略项需更新

### 7.3 安全注意事项

- Git Token 仅用于运维，不泄露到公开位置
- 无管理后台（hexo-admin 已移除）
- Cloudflare 默认 HTTPS + DDoS 防护
- 定期检查 npm 依赖漏洞
